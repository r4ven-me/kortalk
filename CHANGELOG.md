# Changelog

## 0.4.0 — 2026-07-17

### Исправления

- **Критическое: диалог настроек перемешивал провайдеры и промпты.**
  При переключении элементов в списках данные нового элемента записывались
  в предыдущий (неверный порядок обработчиков `currentItemChanged`).
  Конфиг портился при простом листании списка с последующим «Сохранить»,
  включая перенос API-ключа в чужой провайдер.
- Конфиги, испорченные этим багом, автоматически распознаются при запуске:
  секция провайдеров сбрасывается к значениям по умолчанию с предупреждением
  (API-ключи нужно задать заново).
- Закрытие старого popup больше не сбрасывает ссылку на новый popup.
- Автозапуск пишет в `.desktop`-файл абсолютный путь к `kortalk`
  (иначе при установке через pipx `~/.local/bin` может отсутствовать
  в PATH на этапе входа в сессию).

### Изменения

- **Проект переименован: crow-ai → kortalk** (семейство korvus: korserver,
  korctl, kortalk). Новые имена: команда `kortalk`, пакет PyPI `kortalk`,
  конфиг `~/.config/kortalk/config.yaml`, логи `~/.local/state/kortalk/`.
  Прежнее имя crow-ai нигде не используется; конфиг со старого пути
  не подхватывается — перенесите `config.yaml` вручную при необходимости.
- **Интерфейс на двух языках: английский (по умолчанию) и русский.**
  Переключение — Настройки → Общие → Language (полностью применяется после
  перезапуска). Дефолтные промпты и провайдеры для новых установок — на
  английском; CLI-справка и логи — только английские.
- Файл конфига создаётся с правами `600` — в нём могут храниться API-ключи.
- Логирование в `~/.local/state/kortalk/kortalk.log` (ротация 1 МБ × 3);
  `--debug` дублирует лог в stderr.
- Лимит токенов ответа для Anthropic API настраивается
  (Настройки → Общие → «Макс. токенов ответа», по умолчанию 64000).
- Тесты (pytest + pytest-qt), линтер (ruff) и CI (GitHub Actions,
  Python 3.9 и 3.13); публикация на PyPI по тегу `v*` через trusted
  publishing.
